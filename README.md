<details>
<summary>⚙️ <strong>Setup: Enable the Contribution Snake (GitHub Actions)</strong></summary>

<br/>

Create the file `.github/workflows/snake.yml` in your profile repository with:

```yaml
name: Generate Snake Animation

on:
  schedule:
    - cron: "0 */12 * * *"   # runs every 12 hours
  workflow_dispatch:
  push:
    branches:
      - main

jobs:
  generate:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - name: Generate GitHub Contribution Snake
        uses: Platane/snk/svg-only@v3
        with:
          github_user_name: ${{ github.repository_owner }}
          outputs: |
            dist/github-snake.svg
            dist/github-snake-dark.svg?palette=github-dark

      - name: Push to output branch
        uses: crazy-max/ghaction-github-pages@v3.1.0
        with:
          target_branch: output
          build_dir: dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

</details>

<details>
<summary>⚙️ <strong>Setup: Enable Advanced Metrics (metrics.lecoq.io)</strong></summary>

<br/>

For fully automated real-time deep metrics, create `.github/workflows/metrics.yml`:

```yaml
name: Update GitHub Metrics

on:
  schedule:
    - cron: "0 */6 * * *"   # every 6 hours
  workflow_dispatch:

jobs:
  github-metrics:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: lowlighter/metrics@latest
        with:
          token: ${{ secrets.METRICS_TOKEN }}
          user: luizFelippedev
          template: classic
          base: header, activity, community, repositories, metadata
          config_timezone: America/Sao_Paulo
          plugin_isocalendar: yes
          plugin_isocalendar_duration: full-year
          plugin_languages: yes
          plugin_languages_indepth: yes
          plugin_languages_limit: 8
          plugin_languages_sections: most-used, recently-used
          plugin_topics: yes
          plugin_topics_limit: 20
          plugin_stars: yes
          plugin_stars_limit: 4
```

> Create a fine-grained GitHub token with repo read permissions and add as `METRICS_TOKEN` secret.

</details>
